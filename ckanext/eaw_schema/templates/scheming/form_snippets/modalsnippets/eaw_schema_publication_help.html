Publication Package

Set this to "NO" if this package does not supplement a specific publication.

Set this to "YES" if:

1. The package supplements a specific publication (= is a "publication package")
AND
2. The resource you are about to add is that specific publication.

Each Eawag publication should have exactly one data package associated with it (in case that makes sense at all).

Each data package is associated with exactly one publication.

If this switch is YES, the resource should be:

  1. the URL that represents the DOI name of the publication, e.g.
    http://dx.doi.org/10.1002/ecs2.1319
    or, if that is not available,
     
  2. the URL that links to the publication in the Eawag publication repository, e.g.
    http://www.refworks.com/refworks2/?site=039241152255600000%2fRWWS5A549035%2fEawag+Publications&rn=18351
    or, if that is not available either,
     
  3. the manuscript itself (Upload). Please provide the best version available to you.
